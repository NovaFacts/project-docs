### V2__agregar_rol_y_restricciones.sql (Sprint 1)

```sql
CREATE TABLE rol (
    id          INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nombre      VARCHAR(50) NOT NULL UNIQUE,
    descripcion TEXT
);

INSERT INTO rol (nombre, descripcion) VALUES
    ('Administrador',    'Control total del sistema: usuarios, propiedades, configuración'),
    ('Contador',         'Gestión financiera: anticipos, facturas, devoluciones'),
    ('Auxiliar contable','Apoyo contable: registro de reservas y anticipos'),
    ('Recepcionista',    'Atención al cliente: gestión de reservas y consultas');

ALTER TABLE usuario
    ADD CONSTRAINT fk_usuario_rol FOREIGN KEY (rol_id) REFERENCES rol(id);
```

**Effect on data:** The INSERT order determines auto-generated IDs (1 = Administrador, 2 = Contador, 3 = Auxiliar contable, 4 = Recepcionista). Any previously seeded users with `rol_id = 1` or `rol_id = 2` satisfy the FK constraint after V2 runs.

## 3. How the Spring Security Context Handles the New Role

The complete RBAC flow from login to request authorization:

```
POST /api/auth/login
        │
        ▼
UserService.login()
  → Loads User entity (FetchType.EAGER loads Rol automatically)
  → Calls JwtService.generateToken(email, rol.getNombre())
  → JWT payload: { sub: "email", rol: "Administrador", iat: ..., exp: ... }
  → Returns LoginResponse { token, rol: "Administrador", nombre: "..." }
        │
        ▼ (subsequent requests)
JwtAuthenticationFilter.doFilterInternal()
  → Extracts Bearer token
  → Calls JwtService.extractUsername(token)
  → Calls UserDetailsServiceImpl.loadUserByUsername(email)
     → Loads User + Rol (EAGER)
     → Maps role name to authority:
       "Administrador"     → ROLE_ADMINISTRADOR
       "Contador"          → ROLE_CONTADOR
       "Auxiliar contable" → ROLE_AUXILIAR_CONTABLE
       "Recepcionista"     → ROLE_RECEPCIONISTA
     → Returns Spring UserDetails with that authority
  → Sets UsernamePasswordAuthenticationToken in SecurityContextHolder
        │
        ▼
SecurityConfig rule evaluation:
  /api/auth/**         → permitAll()
  /api/usuarios/**     → hasRole("ADMINISTRADOR")  ← checks ROLE_ADMINISTRADOR
  anyRequest           → authenticated()
```

**Key design decisions:**
- `FetchType.EAGER` on `@ManyToOne Rol` ensures the role is always available without lazy-load issues in the filter.
- `hasRole("ADMINISTRADOR")` is equivalent to `hasAuthority("ROLE_ADMINISTRADOR")` — Spring prepends the prefix automatically.
- The role name is also embedded in the JWT payload (`"rol"` claim) so the frontend can read it without an extra API call.

## 7. Manual Testing Steps for RBAC Verification

### 7.1 Seed the database (if not already done)

```bash
# Start the database
cd project-backend && docker compose up -d

# Start with dev profile to trigger seeder
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev
```

The seeder creates user `admin` (Administrador) and `maria` / `juan` / `recepcion` (Contador).

### 7.2 Login as Administrador

```bash
curl -s -X POST http://localhost:8082/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin","password":"admin123"}' | python3 -m json.tool
```

Expected response:
```json
{
  "token": "<jwt>",
  "rol": "Administrador",
  "nombre": "Administrador"
}
```

Save the token: `TOKEN=$(curl ... | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")`

### 7.3 Administrador can access /api/usuarios

```bash
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:8082/api/usuarios | python3 -m json.tool
```

Expected: `200 OK` with user list. Each user shows `nombre`, `email`, and nested `rol` object.

### 7.4 Administrador can access /api/roles

```bash
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:8082/api/roles | python3 -m json.tool
```

Expected: `200 OK` with 4 roles: Administrador, Contador, Auxiliar contable, Recepcionista.

### 7.5 Contador is REJECTED from /api/usuarios (403)

```bash
CONTADOR_TOKEN=$(curl -s -X POST http://localhost:8082/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"maria","password":"maria123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

curl -s -o /dev/null -w "%{http_code}" \
  -H "Authorization: Bearer $CONTADOR_TOKEN" \
  http://localhost:8082/api/usuarios
```

Expected: **`403`**

### 7.6 Unauthenticated request is REJECTED (401)

```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8082/api/roles
```

Expected: **`401`**

### 7.7 Create a new user via Administrador token

```bash
curl -s -X POST http://localhost:8082/api/usuarios \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"nombre":"Ana Martínez","email":"ana@novafacts.com","password":"ana123456","rolId":3}' \
```
